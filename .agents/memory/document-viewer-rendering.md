# In-app document viewer rendering

**Decision:** course-material docs are shown in-app through a single **pdf.js** viewer. Office formats (docx/pptx/…) are converted to PDF server-side with **headless LibreOffice** and cached; PDFs are served directly; txt/md as text.
**Why:** there is no reliable pure-JS PowerPoint renderer, so LibreOffice→PDF is the only path to high-fidelity slide rendering. It runs headless in the Replit sandbox and carries into deploys.

**Gotchas worth remembering (not obvious from code):**
- LibreOffice headless startup prints `Could not find a Java Runtime Environment` and `Failed to initialize OpenCL` — these are **harmless**; conversion still succeeds. Don't chase them.
- Concurrent `soffice` runs clash on the user profile — each conversion needs its own `-env:UserInstallation` dir and conversions must be serialized.
- **pdfjs-dist v5** worker must be loaded via Vite `?url` from the `.mjs` worker. The older cdnjs `pdf.worker.min.js` pattern elsewhere in the repo is stale for v5 (v5 ships `.mjs`) — don't copy it.
- **The "not an error object" overlay is owned by Replit's preview-iframe harness (`injected.js`), NOT a Vite plugin.** It is injected ahead of app code, so app-side `window` listeners always register *after* it. `preventDefault()` is therefore **best-effort only** — you cannot win the registration-order race, so do not rely on suppression as "the fix." The reliable lever is **diagnostics + removing the throw at source.**
- **Many leaked reasons are NOT even error-shaped.** Earlier guards keyed on "error-shaped non-Error" (`instanceof Error`, or plain object with string `name`+`message`) — pdf.js `BaseException` (`prototype = new Error()`) matches that, but the crash kept recurring and the guard's `console.warn` **never appeared in browser logs**, proving the real reason failed that predicate. That points to a value with **no `name`** (Supabase `PostgrestError` = `{message,details,hint,code}`), or `null`/`undefined`/a bare string. So the only safe detector is Replit's own exact criterion: `Object.prototype.toString.call(reason) !== '[object Error]'` — nothing narrower.
- **When you can't reproduce, instrument first.** A global guard that **logs full diagnostics** for every non-Error reason (type, toString-tag, ctor, name, message, code, status, `Object.keys`, stack≤1000, `[async-guard]` prefix) is the fastest path: the browser console is captured by `refresh_all_logs`, so one repro reveals the exact culprit+stack even though the guard runs after `injected.js`.
- **Fix location is GLOBAL, not viewer-scoped.** A single early guard in `src/main.tsx` (registered **before `createRoot`**, capture phase) for both `unhandledrejection` and `error` supersedes any per-component listener. Skip `error` events with an element `target` (resource-load failures). Keep the React error boundary too (render-phase throws). Broad suppression of all non-Error reasons is acceptable *because* every one is logged in full — narrowing the predicate just reintroduces the gap that let the real reason slip through.
- **RESOLVED root cause of the viewer crash: a `ResizeObserver` feedback loop, NOT pdf.js.** The `[async-guard]` diagnostics captured the exact reason: a bare **string** `"ResizeObserver loop completed with undelivered notifications."` on a window `error` event. The viewer's `ResizeObserver` callback called `renderPage()` synchronously, which resizes the canvas and retriggers observation in the same delivery cycle. **Fix:** defer the callback to `requestAnimationFrame` (single cancel+reschedule rafId) AND only re-render when the rounded container *width* actually changed (init `lastWidth` from `clientWidth` so the immediate observe callback is a no-op). This is the canonical fix for that warning; do not reach for suppression first.
- Symptom note: PPTX viewer crash ~30–65s after click is LibreOffice conversion finishing (dev StrictMode double-mounts ⇒ ~2× conversion time); the delay is the conversion. The crash itself, though, was the ResizeObserver loop above (the resize storm happens as the split-view/canvas settles), not a cancelled-render rejection.
